# Relaciones de 01_titulo_vii_programas_de_ordenador

## Función del bloque

Este fragmento concentra el régimen especial del software dentro de la ley y define la secuencia lógica completa: objeto protegido, titularidad, explotación, límites, registro, infracción y protección.

## Relaciones internas del bloque

- La originalidad delimita qué programa queda protegido.
- La titularidad determina quién puede explotar o autorizar el uso.
- Los derechos de explotación concretan el contenido económico del control.
- Los límites al derecho de explotación restringen ese control en favor del usuario legítimo.
- La infracción aparece cuando se vulneran esos derechos fuera de los límites legales.
- Las medidas de protección activan la defensa del titular frente a la infracción.

## Enlaces conceptuales útiles

- Programa de ordenador -> se protege si -> es original.
- Protección del software -> recae sobre -> forma de expresión.
- Ideas y principios -> no forman parte de -> protección autoral.
- Programa creado por trabajador -> atribuye explotación a -> empresario.
- Usuario legítimo -> puede realizar -> actos necesarios para el uso.
- Usuario legítimo -> puede observar y estudiar -> funcionamiento del programa.
- Descompilación -> solo se permite para -> interoperabilidad.
- Interoperabilidad -> no autoriza -> copia sustancialmente similar.
- Infracción -> habilita -> acciones y medidas cautelares.
- Régimen del software -> convive con -> patentes, marcas, secretos comerciales y competencia desleal.
